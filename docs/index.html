api-puppy

This is not another framework or library or even a template, it's just a demo!
A fun project to play with different ways to use Web APIs

Project Documentation

You are going to find bugs because this is just a DEMO! Lots of work needs to be done and things may or may not get worked on. Use Google Chrome and you will get less errors in the DEBU UI.

Have fun with this code. Clone the project and host it yourself. It's easy to learn by breaking things and this is the project to do it in.

A working server(https://demo.tektology.com/) is up and running most of the time, but it's my server that I have to pay for so it may or may not be up at any given time.

Folder Structure

This is a quick breakdown of the top level folders. Run the "SETUP.js" file in the root of the project to ensure you have the required folders needed for a basic install.

SECRET

You must create a folder called SECRET in the root of the project!

This folder is used put logs in and various configuration stuff that nobody wants to see in the project source.

CERTS

Your SSL CERTS go in this folder. You must make sure you create your own CERTS folder and put your own CERTS in them.

** UPDATE **
CERTS is not in the SECRET folder! :-)

APIServer

This folder has all the server side script files you need to provide the web API over a web server.

Docs

GitHub sites uses this folder to provide a free website for documentation.

Notes

Find helpful information about running this project and things probably forgotten over time that should be written down.

Tools

Operating the server is easier if you script common tasks out. Both BASH and JS files can be used.

FAQ

Frequently Asked Questions

Is this yet another framework or library?

NO!

Absolutly not no please no not another framework or library!

Why not just use express and other 3rd party middleware?

Why?

Why should I add extra code thats not needed? Because it's easier? I think not. :-)

The best part about this whole project is the lack of NPM or 3rd party influence. This is pure JavaScript, Node, and of course some HTML and CSS for the debugger thing.

Your code looks like spaghetti cooked by a chaotic evil newbie!

Yep

I'm just having fun here and trying to reach the lowest level of user. But feel free to fork, clone, or peel off what ya want and do a better job. I'd love to see other peoples ideas of improvement.

Why use the name "api-puppy"?

Everyone loves puppies!

I wanted to choose a name that nobody could hate. I feel like tech these days use marketing way too much but I can't beat them so I join them. A cheap jedi mind trick to ensure that the name will most likely be at least liked if not loved and who does not love a puppy?

Where are the async and promises?

Where they belong

I promise to use a promise if and when it's needed.

Remember this is just a demo and not production code. If you want to use different patterns and practices then please do so. Fork the project and make any improvements you like.

APIServer.js

The APIServer.js file is the master script to run on your local machine to play with the code.

Change the port number and host name (if you plan on using https) then open your browser to the address listed in the console. By default, it's http://0.0.0.0:9080

API Services

It's easy to add a new service.

The services folder is where you put your new folder that is, in fact, your service. An easy way to get started is to just copy the default folder and start from there.
